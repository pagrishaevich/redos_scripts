# Contributing

Спасибо за вклад в `redos_scripts`.

## Как предложить изменения

1. Сделайте fork репозитория.
2. Создайте отдельную ветку для изменений.
3. Внесите правки и проверьте скрипты локально.
4. Откройте Pull Request с описанием цели изменений.

## Рекомендации по изменениям

- Сохраняйте совместимость с РЕД ОС 8.
- Не добавляйте секреты и учетные данные в репозиторий.
- Для shell-скриптов проверяйте синтаксис через `bash -n`.
- По возможности придерживайтесь существующего стиля и структуры скриптов.

## Что указывать в Pull Request

- Что изменено и зачем.
- На каких сценариях/командах проверено.
- Какие есть ограничения, риски или требования по правам root.
